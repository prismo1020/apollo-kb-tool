How to Use This Portal

Apollo is Sandbox VR's internal AI agent, powered by a library of Knowledge Base (KB) files uploaded to Chatbase. When Apollo gives a wrong answer, this portal is how we fix it — submit the bad answer, let the AI editor draft a clean KB rewrite, review and approve it, then push the updated file back to Chatbase. No manual file editing required.

The Workflow — in order
  1. Submit — Paste the question, Apollo's wrong answer, and the correct guidance. Add the Oasis SOP link if the topic has one. Hit Submit Single or queue up several and hit Submit Batch.
  2. Wait for Analysis — The Apollo KB Editor bot (a dedicated Chatbase agent) reads the correction and drafts a clean rewrite of the relevant KB section. This runs automatically every 10 minutes, or hit Run Automation Now to trigger it immediately.
  3. Review — Go to the Review Queue tab. Corrections marked analysis ready are ready to approve. Ones marked needs review need a closer look — usually a missing Oasis link or low confidence match.
  4. Approve or Reject — Read the before/after diff. Edit the proposed replacement if needed. If it looks good, click Approve Update. If it's wrong, add a rejection reason and click Reject — you can resubmit with better info.
  5. Apply — Approved corrections are picked up automatically and written to the KB file in GitHub. The correction flips to applied.
  6. Push to Chatbase — Once applied, use Copy for Chatbase or Download File to get the updated file, then upload it to Chatbase (see below).
Tips — Writing Good Corrections
  • Paste Apollo's exact wrong answer — the more specific, the better the AI editor's rewrite.
  • Approved guidance should be what Apollo should say, written the way you'd want to see it in the KB. The AI will use your words as a starting point.
  • Always include an Oasis SOP link for any procedural topic. Corrections without one get flagged needs review and can't be auto-approved.
  • Keep one correction per issue. Don't combine two unrelated problems into one submission.
  • Add reviewer notes for edge cases — e.g. "This only applies to TactSuit Pro, not X40."
Tips — Batching Corrections
  • Use Add to Batch to queue multiple corrections, then submit them all at once. Great for post-stress-test cleanup sessions.
  • Batch by category when possible — all TECH corrections together, all OPERATIONS together — so the review queue stays organized.
  • Review batch corrections together in one sitting. It's faster to approve 8 corrections in a row than to come back to each one individually.
Updating Chatbase — file by file
  • In the Review Queue, find an applied correction and click Copy for Chatbase. This copies the full updated KB file text to your clipboard.
  • Go to Chatbase → Sources → Files.
  • Search for the file using its number — e.g. type _70_ to find file 70.
  • Delete the old file, upload the new one, then click Retrain.
  • Do this in batches for consistency — update all your corrections before retraining.
Exporting the Full KB
  • Go to the Activity Log tab to find the export buttons.
  • Download Full KB (ZIP) — downloads every KB file as a `.txt`, zipped. Use this to retrain Apollo from scratch, back up the full catalogue, or share the current KB with someone.
  • Download Merged KB (single file) — all KB files concatenated into one `.txt`. Perfect for training a Gemini Gem, generating patch notes, or giving a new team member a full picture of what Apollo knows.
  • Both exports pull the latest committed versions from GitHub — always up to date with approved corrections.
